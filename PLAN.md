# Dotnetstore.DocumentViewer

## Introduction
Dotnetstore.DocumentViewer should be a secure way of viewing documents in an Avalonia application. 
It should be able to handle a variety of document formats and provide a user-friendly interface for viewing them.
An user should be able a document without being able to edit it, copy it, save it, or print it. 
The document should be rendered in a way that is visually appealing and easy to read.

## Features
- Support for multiple document formats (PDF, DOCX, etc.)
- Secure viewing (no editing, copying, saving, or printing)
- User-friendly interface
- Visually appealing rendering
- Easy administration of document access rights
- Integration with authentication and authorization systems

## Implementation
The implementation of Dotnetstore.DocumentViewer will involve the following steps:
1. Research and select appropriate libraries for rendering different document formats in Avalonia.
2. Design and implement the user interface for the document viewer.
3. Implement security measures to prevent editing, copying, saving, and printing of documents.
4. Integrate the document viewer with authentication and authorization systems to manage access rights.
5. Test the document viewer with various document formats to ensure compatibility and performance.
